# Product Quality Rubric

Use this rubric to score product work before review. A score of 1 blocks the
relevant flow. A score of 5 is the world-class target.

## Save Loop

1. Save does not create a review item, or creates inconsistent local storage.
2. Save creates a record but misses source, duplicate handling, or review state.
3. Save creates saved word and initial review state for approved sources.
4. Save preserves progress, handles duplicates, and clearly routes to review.
5. Save feels instant, trustworthy, and naturally moves the learner into recall.

## Review Loop

1. Answers do not persist events or state.
2. Answers persist partial data but cannot support Due, Weak, or Mastered.
3. Answers create review events and update review state.
4. Sessions support due, weak, focused, and pack preview review without state
   conflicts.
5. Review feels short, focused, and educational while preserving complete event
   history.

## SRS Truthfulness

1. Box, mastery, due date, or weak score is fake or random.
2. State changes are mostly real but can mark mastery from shallow recall.
3. State follows the VLX 5-box rules and delayed recall requirement.
4. Slow, guessed, wrong, and repeated-mistake paths are handled honestly.
5. The learner and reviewer can audit why each memory state changed.

## Due/Weak/Mastered

1. Counts or lists are fabricated.
2. Counts exist but mix mock and live state in misleading ways.
3. Counts derive from local review state and due dates.
4. Empty, new, due, weak, and mastered states are clear and actionable.
5. The dashboard makes the next best learning action obvious from true state.

## Pack Progress

1. Pack progress is invented or unrelated to review events.
2. Preview start or completion is recorded inconsistently.
3. Preview progress records reviewed and correct counts from review sessions.
4. Pack detail, catalog, and review completion stay synchronized.
5. Pack progress helps a learner choose the next review without overstating
   unlocks, completion, or paid access.

## Paywall Clarity

1. UI creates or implies real payment when no payment is connected.
2. CTAs are present but beta/payment status is unclear.
3. Lite and Pro CTAs capture local or configured external interest honestly.
4. Paywall prompts appear at learning-relevant moments with clear outcomes.
5. Monetization feels premium, transparent, and secondary to the memory loop.

## Alias/Search Safety

1. Unknown aliases create fake saves or unsafe routes.
2. Known aliases work but empty or unknown states are confusing.
3. Known aliases resolve to canonical slugs and unknown aliases have no action.
4. Alias saves preserve source and create the same review state as word pages.
5. Search helps learners safely move from encountered language to English
   memory cards without creating false content.

## Weak Sprint

1. Weak sprint uses fake weak words or does not update state.
2. Weak sprint exists but has unclear selection or event behavior.
3. Weak sprint selects real weak words and writes review events/state.
4. Empty and completion states are honest and useful.
5. Weak sprint feels like a focused rescue path for fragile memory.

## Dashboard Usefulness

1. Dashboard is decorative or shows fake metrics.
2. Dashboard contains useful data but review is not the primary action.
3. Dashboard prioritizes Today Memory Mission with real due and weak counts.
4. Saved library, pack progress, and paywall prompts support the memory mission.
5. The dashboard makes Weekly Reviewed Words more likely every time it loads.

## Premium/Calm UI

1. UI feels noisy, childish, cluttered, or untrustworthy.
2. Visual style is usable but inconsistent or overdramatic.
3. UI is calm, minimal, readable, and learning-focused.
4. Hierarchy, spacing, copy, and states support repeat study sessions.
5. The experience feels warm, credible, premium, and restrained without hiding
   important learning actions.
